# 职途AI 产品体检

本文件记录最后一轮产品体检结论，便于项目复盘、公开展示和后续迭代。

## 1. 页面结构

现状：

- 主导航分为项目总览、简历实验室、面试训练场、投递看板、AI 教练，信息架构已经清晰。
- 每个业务页内部使用二级功能导航，避免所有功能堆在一屏。
- Anime / Glass 双主题已经统一到 Logo、背景图、卡片、按钮和素材切换。

已补强：

- 首页新增“今日求职作战台”，把数据指标、短板和下一步行动集中展示。
- 投递看板的跟进建议回到当前模块展示，避免用户点了按钮却看不到反馈。
- 新增独立项目展示页 `static/showcase.html`，用于 GitHub Pages 展示和项目介绍。

## 2. 功能闭环

现状：

- 简历可以录入、上传、编辑、分析、优化、导出。
- JD 可以反向驱动简历优化和模拟面试。
- 面试训练覆盖模拟流程、题库练习、专业知识面试、录音分析和记录复盘。
- 投递看板能记录、推进、生成跟进话术。

已补强：

- 首页根据真实数据生成求职准备度，提示“下一步该做什么”。
- 训练记录可查看详情和删除，形成练习后的复盘闭环。
- 对“不知道 / 下一题 / 跳过”等回答做兜底判断，减少 AI 幻觉式评分。

后续可扩展：

- 增加日历提醒和面试时间提醒。
- 增加简历版本 diff 对比。
- 增加投递转化率漏斗图。

## 3. 数据流

现状：

- SQLite 存储简历、JD 匹配、面试、练习、录音、投递数据。
- 前端通过统一 `api()` 封装访问后端。
- 上传文件、导出文件、运行数据已通过 `.gitignore` 排除，不进入 GitHub。

已补强：

- `/api/dashboard/<user_id>` 增加 `career_pulse`，用于输出求职准备度、短板和行动计划。
- `/api/training-records/<user_id>` 返回面试 conversation，支持复盘完整对话。
- `/api/uploads/<filename>` 支持训练记录中播放已保存录音。

后续可扩展：

- 将用户 ID 抽象为登录态。
- 把 AI 调用记录和 token 消耗记录入库。
- 增加数据导出为 JSON / CSV 的功能。

## 4. 后端智能体能力

现状：

- 多模型网关支持 GLM、DeepSeek、Kimi / Moonshot。
- 未配置 API Key 时有本地规则兜底。
- 简历诊断、JD 匹配、面试反馈、题库评分都具备结构化输出。

已补强：

- AI 教练可以读取当前用户上下文：简历、投递、面试训练。
- 求职作战报告把简历资产、岗位匹配、面试训练、投递推进串起来。
- 音频分析结合真实声音指标和文本内容，而不是只做文字评分。

后续可扩展：

- 对不同岗位维护提示词模板。
- 增加“面试官追问强度”配置。
- 增加面试录音自动转写服务接入。
