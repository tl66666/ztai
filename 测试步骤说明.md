# 测试步骤说明

本文档用于本地功能体验、项目演示和回归测试。

## 1. 启动项目

```powershell
cd "C:\Users\唐乐\Desktop\实训\项目\jobhunter"
python -m pip install -r requirements.txt
python app.py
```

浏览器访问：

- 主系统：<http://localhost:5000>
- 项目展示页：<http://localhost:5000/showcase.html>

## 2. 页面体验清单

### 项目总览

- [ ] 左侧品牌显示“职途 AI”。
- [ ] Anime / Glass 两套主题可以切换。
- [ ] Hero 图片完整显示，不遮挡文字和按钮。
- [ ] 职业脉冲卡片显示简历、面试、JD 匹配、投递状态。
- [ ] 点击“项目展示”可以进入展示页。

### 简历实验室

- [ ] 可以上传 Word / PDF 简历或粘贴文本保存简历。
- [ ] “我的简历”可以查看、编辑、删除记录。
- [ ] 编辑简历时可以打开可编辑面板，并保留原始文件入口。
- [ ] 简历分析可以选择指定简历，输出优势、风险、证据链和修改建议。
- [ ] JD 优化可以粘贴岗位描述，也可以通过 Boss 直聘入口获取 JD 后再复制回来分析。
- [ ] 技能图谱可以基于所选简历生成技能覆盖、短板和补强路线。
- [ ] 导出与格式转换支持 PDF、Word、PDF 转 Word、Word 转 PDF。

### 面试训练场

- [ ] 模拟真实面试点击开始后进入训练态，而不是停留无反馈。
- [ ] 面试流程包含自我介绍、项目深挖、技术追问、行为面、反问总结。
- [ ] 语音回答可以录音、回放、转写，并生成语速、停顿、语气词等反馈。
- [ ] 专业知识面试可以按岗位方向生成问题和追问。
- [ ] 题库练习可以答题、查看解析、跳过下一题。
- [ ] 训练记录可以查看回答文本、语音信息、AI 反馈，也可以删除。
- [ ] 用户回答“不知道”“下一题”“跳过”时，系统能给出合理反馈或切题跳转。

### 投递看板

- [ ] 可以新增投递记录，记录公司、岗位、阶段、城市和备注。
- [ ] 看板可以按投递阶段查看记录。
- [ ] 薪资评估和投递复盘有明确入口。

### AI 教练

- [ ] 可以围绕求职策略、简历优化、面试表达进行问答。
- [ ] 没有 API Key 时会进入本地兜底，不会阻塞页面。

## 3. 后端接口检查

推荐运行自动化测试：

```powershell
python tests\test_core_features.py
```

核心接口：

- `GET /api/dashboard/1`
- `GET /api/resumes/1`
- `POST /api/resumes`
- `POST /api/resumes/<id>/analyze`
- `POST /api/resumes/<id>/jd-optimize`
- `POST /api/interview/start`
- `POST /api/interview/answer`
- `GET /api/training-records`
- `DELETE /api/training-records/<id>`
- `POST /api/applications`
- `POST /api/config/model`

## 4. GitHub 提交前检查

```powershell
git status --short --ignored
git diff --check
rg -n "sk-|api[_-]?key|API_KEY|token|secret|password" . --glob "!uploads/**" --glob "!exports/**" --glob "!output/**" --glob "!__pycache__/**" --glob "!*.db"
```

确认以下内容没有被提交：

- `.env`
- `jobhunter.db`
- `uploads/`
- `exports/`
- `output/`
- 个人真实简历文件

## 5. 推荐体验顺序

1. 打开项目展示页，了解项目定位、技术栈和智能体能力。
2. 回到主系统，展示主题切换和模型配置。
3. 在简历实验室上传或选择一份简历，体验简历分析、JD 优化和技能图谱。
4. 进入面试训练场，体验模拟面试、语音回答、AI 反馈和训练记录。
5. 打开投递看板，体验投递记录和复盘能力。
6. 最后展示 README、产品体检文档和自动化测试结果。
